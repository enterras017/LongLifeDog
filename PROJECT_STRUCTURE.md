# LongDogLifeNew プロジェクト構造

## プロジェクト概要
- **メインアプリ**: ながいぬのいる生活（ご飯あげて長いぬを伸ばすメイン機能）
- **サブゲーム**: ご飯ランナー（スワイプで移動するミニゲーム）

## アプリ構成

### メイン画面（LongDog.tsx）
- **役割**: アプリのメイン機能
- **機能**: 
  - ながいぬにご飯をあげる
  - ながいぬの体が伸びる
  - なでなで機能
  - リセット機能
- **アクセス**: アプリ起動時のデフォルト画面
- **ボタン**: 右上に「🏃‍♂️」ボタンでサブゲームにアクセス

### サブゲーム画面（FoodRunner.tsx）
- **役割**: ミニゲーム
- **機能**:
  - スワイプでながいぬを移動
  - ご飯を食べてスピードアップ
  - 壁や自分にぶつかったらゲームオーバー
- **アクセス**: メイン画面の右上ボタンから
- **戻る**: 左上の「← メインに戻る」ボタン

## ファイル構成
```
App.tsx                    # メイン/サブゲーム切り替え
src/components/
├── LongDog.tsx           # メイン画面（ご飯あげ機能）
├── SnakeGame.tsx         # サブゲーム（ご飯ランナー）
├── LongDogHead.tsx       # ながいぬの頭
├── LongDogBody.tsx       # ながいぬの体
└── LongDogTail.tsx       # ながいぬの尾
```

## 開発ルール
1. **メイン機能が優先**: ご飯あげ機能がメイン、ゲームはサブ
2. **画面切り替え**: App.tsxで状態管理
3. **戻る機能**: サブゲームからは必ずメインに戻れる
4. **UI統一**: メイン画面のデザインを基準とする

## 今後の開発方針
- メイン機能の改善が最優先
- サブゲームはメイン機能を補完する位置づけ
- 新機能追加時はメイン/サブの役割を明確に分ける
