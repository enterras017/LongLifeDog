### 2. APP_SPECIFICATION.md ⭐
**対象者**: プロダクトマネージャー、デザイナー、企画、全開発者  
**ページ数**: 約50ページ  
**読了時間**: 30-40分  

**内容**:
- **アプリ概要**: コンセプト、ターゲットユーザー
- **機能仕様**: 全機能の詳細な説明
  - メイン画面の仕様
  - サブゲームの仕様
  - UI/UX設計
- **ゲームバランス**: スピード調整、難易度曲線
- **データ仕様**: 現在と将来のデータ構造
- **デザイン仕様**: カラー、タイポグラフィ、レイアウト
- **パフォーマンス要件**
- **対応環境**
- **ユーザーストーリー**: 3つの具体的な使用例
- **成功指標（KPI）**
- **リスクと対応策**
- **FAQ**: ユーザー向け、開発者向け

**おすすめの読み方**:
1. 第1-2章: アプリの全体像を理解
2. 第1-2章の機能仕様: 自分が関わる部分を詳しく読む
3. 第10章: ユーザーストーリーで使用感をイメージ
4. 必要に応じて他の章を参照

**ChatGPTに共有する場合**:
- ✅ アプリの機能について質問したい
- ✅ ユーザー体験について相談したい
- ✅ 機能追加のアイデアを議論したい

### 3. TECHNICAL_DOCUMENTATION.md ⭐
**対象者**: エンジニア、技術リード  
**ページ数**: 約100ページ  
**読了時間**: 60-90分（熟読）/ 20-30分（流し読み）

**内容**:
- **詳細な技術仕様**: コード例付き
  - メイン機能の完全な仕様
  - サブゲームの詳細なゲームロジック
  - 状態管理の設計
- **開発ルールと設計原則**:
  - コンポーネント設計
  - 状態管理のベストプラクティス
  - アニメーション設計
  - エラーハンドリング
  - TypeScript使用ルール
  - パフォーマンス最適化
- **解決済みエラーと再発防止策**: 6つの主要エラー
  1. ネイティブモジュールリンクエラー
  2. スワイプ操作が反映されない問題
  3. ゲームが即座にゲームオーバーになる問題
  4. React Nativeバージョンミスマッチ
  5. iOSビルドエラー
  6. 白い画面問題
- **今後の技術的課題**: 7つの分野
  1. パフォーマンス最適化
  2. TypeScript型定義の厳密化
  3. 状態管理の改善
  4. テストの実装
  5. エラー監視とクラッシュレポート
  6. アクセシビリティ
  7. ネットワーク機能
- **運用上の検討課題**: 9つの分野
  1. データ管理戦略
  2. バージョン管理
  3. アプリストア申請準備
  4. マーケティングとプロモーション
  5. メンテナンスとサポート
  6. 収益化戦略
  7. 法的・コンプライアンス
  8. グローバル展開
  9. 長期的なビジョン

**おすすめの読み方**:
1. **初回**: 第1章（アプリケーション詳細仕様）を熟読
2. **開発前**: 第2章（開発ルール）を確認
3. **エラー発生時**: 第3章（解決済みエラー）で類似事例を探す
4. **機能追加時**: 第4章（今後の技術的課題）で参考実装を確認
5. **運用計画時**: 第5章（運用上の検討課題）を参照

**特に重要なセクション**:
- **スワイプ操作の実装** (第1章): 最も複雑な実装の1つ
- **エラー2: スワイプ操作が反映されない** (第3章): トラブルシューティングの実例
- **パフォーマンス最適化** (第4章): 将来の課題

**ChatGPTに共有する場合**:

- ✅ エラーのトラブルシューティング
- ✅ アーキテクチャの改善相談

## 🔍 シーン別ドキュメント活用ガイド

### シーン1: 新しいメンバーがジョインした
```
1. README.md で全体像を把握（5分）
2. PROJECT_STRUCTURE.md でプロジェクト構造を理解（5分）
3. APP_SPECIFICATION.md でアプリ仕様を学習（30分）
4. TECHNICAL_DOCUMENTATION.md で技術詳細を確認（60分）
```

### シーン2: 機能追加の相談をしたい
```
1. APP_SPECIFICATION.md で既存機能を確認
2. 該当する「将来の拡張計画」を確認
3. TECHNICAL_DOCUMENTATION.md で技術的な実装方法を確認
4. ChatGPTに両ドキュメントを共有して相談
```

### シーン3: エラーが発生した
```
1. TECHNICAL_DOCUMENTATION.md の第3章（解決済みエラー）を確認
2. 類似エラーがあれば、その解決手順を試す
3. なければ、第2章（開発ルール）でアンチパターンを確認
4. ChatGPTにTECHNICAL_DOCを共有してトラブルシューティング
```

### シーン4: ユーザー体験を改善したい
```
1. APP_SPECIFICATION.md の第10章（ユーザーストーリー）を読む
2. 第5章（UI/UXデザイン）を確認
3. 第12章（リスク）でユーザー体験に関するリスクを確認
4. ChatGPTにAPP_SPECを共有して改善案を相談
```

### シーン5: リリース準備をしたい
```
1. APP_SPECIFICATION.md の第7章（対応環境）を確認
2. TECHNICAL_DOCUMENTATION.md の第5章（運用課題）を熟読
3. 特に「アプリストアへの申請準備」セクションを確認
4. チェックリストを作成
```

## 🆘 困ったときは

### Q: どのドキュメントを読めばいいかわからない
**A**: まず README.md を読んでください。そこから他のドキュメントへのリンクがあります。

### Q: ドキュメントが長すぎて全部読めない
**A**: 目次を見て必要な部分だけ読んでください。全部読む必要はありません。

### Q: ChatGPTに何を共有すればいいかわからない
**A**: 相談内容に応じて：
- アプリの機能 → APP_SPECIFICATION.md
- 技術的な実装 → TECHNICAL_DOCUMENTATION.md
- 両方必要な場合 → 両方共有

### Q: ドキュメントに書いていないことがある
**A**: 新しい発見や解決策は、該当するドキュメントに追記してください。
